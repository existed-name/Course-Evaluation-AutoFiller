### 🚀 针对“模块 2”的 AI 提示词

**角色设定：**
你是一位资深的自动化测试架构师。请基于 Java 21 和 Selenium 4.x，协助我完成“教务系统自动评教项目”的第二阶段：**页面导航与科目列表提取**。

**已知前提：**

1. 模块一已完成：`WebDriver` 实例已启动，且通过 `user-data-dir` 绕过了登录，目前浏览器停留在教务系统首页。
2. 技术栈：Java 21, Selenium 4.40.0, EdgeDriver。
3. 目标系统特征：基于 ColdFusion (.cfm)，HTML 元素的 ID 具有动态随机性（如 `id="btn_12345"`）。


#### **【第一部分：实战教程】**

1. **F12 元素定位技巧：** 针对这种含有动态 ID 的老旧教务系统，请教我如何利用 Chrome/Edge 开发者工具（F12）编写稳健的 XPath 或 CSS 选择器（重点讲解 `contains(text(), '...')` 和层级定位）。
2. **显式等待（Explicit Wait）原理：** 解释为什么在点击菜单跳转到列表页时，必须使用 `ExpectedConditions` 而不是 `Thread.sleep()`。

#### **【第二部分：工程化代码实现】**

请根据以下要求编写代码：

**1. 定义数据模型：**

* 创建一个名为 `EvaluationSubject` 的类（或 Java 21 的 `record`），用于封装科目信息。包含：课程名称、教师姓名、状态（待评价/已完成）以及对应的“进入评价”按钮的 `WebElement`。

**2. 编写 `EvaluationNavigator` 类：**

* **方法：** `MapsToEvaluationPage(WebDriver driver)`
* **逻辑：** 使用显式等待寻找并点击“教学质量评价”菜单。考虑到 ID 动态，请优先使用包含文本的 XPath 定位。

**3. 编写 `SubjectExtractor` 类：**

* **方法：** `List<EvaluationSubject> extractPendingSubjects(WebDriver driver)`
* **解析逻辑：** * 精确定位评价列表表格（Table）。
* 遍历表格行（TR），过滤掉“已完成”的科目，仅提取“待评价”的行。
* 将每一行的数据封装进 `EvaluationSubject` 列表。


* **控制台输出：** 打印“成功解析页面，发现 X 门科目，其中 Y 门待评价”。

**4. 核心规范：**

* **常量管理：** 所有的 XPath 表达式必须声明为 `private static final String` 常量。
* **异常处理：** 使用 `try-catch` 包裹定位逻辑，若超时未找到元素，打印当前的 URL 和 Page Title 方便调试。
* **显式等待：** 统一使用 `Duration.ofSeconds(10)` 的超时时间。
